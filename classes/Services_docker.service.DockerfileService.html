    checkAndCreateContainerDiferentPort

    • checkAndCreateContainerDiferentPort(
          containerName: string,
          image: string,
          port: number,
          port2: number,
          volume: string,
          network: string,
          envVars?: string[],
      ): Promise<void>

      Ensures that a Docker container is running, and creates it if it does not exist.

      This method first checks whether a container with the given name is running.

      • If the container is running, it logs a confirmation.
      • If the container exists but is stopped, it starts it.
      • If the container does not exist, it creates and starts it using the provided configuration.

      The method adjusts volume mounting paths based on the container type (e.g., MySQL, MongoDB, etc.) and includes environment variables if provided. It binds a host port to a container port as specified.

      Parameters

      • containerName: string

        The name of the Docker container to check or create.

      • image: string

        The Docker image to use if the container needs to be created.

      • port: number

        The host port to bind.

      • port2: number

        The container port to expose.

      • volume: string

        The volume path to mount in the container.

      • network: string

        The Docker network to which the container should be connected.

      • OptionalenvVars: string[]

        (Optional) An array of environment variables in 'KEY=VALUE' format.

      Returns Promise<void>

      A promise that resolves when the container is verified to be running or created successfully.

      Throws

      An error if Docker commands fail during creation or startup of the container.

      Creates a MariaDB database and user inside a running Docker container.

      This method executes a shell command within the specified MariaDB Docker container to:

      • Create a new database if it does not already exist.
      • Create a new user with the given credentials, if not already present.
      • Grant all privileges on the newly created database to the new user.

      If successful, it returns a MariaDB connection URI using the provided credentials.

      Parameters

      • containerName: string

        The name of the Docker container running MariaDB.

      • dbName: string

        The name of the database to create.

      • dbUser: string

        The username to create and assign privileges.

      • dbPassword: string

        The password for the database user.

      Returns Promise<string>

      A promise that resolves to the MariaDB connection URI.

      Throws

      An error if the command fails to execute or database/user creation fails.

    deleteDataBase

    • deleteDataBase(db_type: any, db_name: any, db_user: any): Promise<void>

      Deletes a database and its associated user from a specified database engine.

      Based on the db_type parameter, this method constructs and executes a Docker command to connect to the appropriate container and run database-specific commands to:

      • Drop the database if it exists.
      • Drop the associated user if it exists.
      • Flush privileges or cleanup as necessary.

      Supported database types:

      • 'N' for MongoDB
      • 'P' for PostgreSQL
      • 'M' for MariaDB
      • Any other value defaults to MySQL

      Parameters

      • db_type: any

        The type of database: 'N' (MongoDB), 'P' (PostgreSQL), 'M' (MariaDB), or default (MySQL).

      • db_name: any

        The name of the database to delete.

      • db_user: any

        The username associated with the database to delete.

      Returns Promise<void>

      A promise that resolves if the database and user were deleted successfully.

      Throws

      An error if the Docker command execution fails or the database/user cannot be deleted.
